Mi trabajo de sueños...

Descripción del trabajo de mis sueños...

Un lugar ideal donde pueda poner en práctica mis habilidades como Fullstack Developer & QA Engineer para contribuir al desarrollo de sistemas de apoyo para el proceso de procuración e impartición de justicia como AROS, una plataforma integral orientada a modernizar el proceso mediante la interconexión y automatización de procesos. Busco formar parte de una organización comprometida con la innovación tecnológica, guiada por sólidos principios éticos y sociales, y conformada por personas colaborativas, comprometidas y con un genuino espíritu de servicio hacia la ciudadanía.

Razones por las que me gustaría tener este trabajo...

  • Los objetivos y principios están alineados con los míos.
  • El área ofrece desarrollo profesional y colaboración en proyectos innovadores.
  • Relación con personas de diversas culturas y puntos de vista enriquecedores.

Requisitos para conseguir el trabajo...

Responsabilidades Principales Requisitos Técnicos (Obligatorios) Deseables (Nice to have) Habilidades blandas
Crear y mantener documentación de pruebas y reportes de ejecución. 2+ años de experiencia comprobable en QA Automation. Experiencia con Cypress, TestCafe o Selenium. Atención al detalle y pensamiento analítico.
Diseñar, desarrollar y mantener pruebas automatizadas de UI y API utilizando Playwright (TypeScript o JavaScript). Experiencia sólida con Playwright (JavaScript o TypeScript). Experiencia en BDD (Cucumber / Gherkin). Comunicación clara con equipos técnicos y no técnicos.
Implementar y ejecutar pipelines de automatización en Azure DevOps. Conocimientos en Azure DevOps Pipelines (configuración, ejecución y monitoreo). Conocimientos básicos de SQL o bases de datos NoSQL. Capacidad para trabajar en equipos ágiles (Scrum / Kanban).
Mantener la integración continua de pruebas en entornos de desarrollo y staging. Experiencia con Git y flujos de trabajo GitFlow o similares. Familiaridad con reportes de pruebas automatizadas (Allure, Playwright Test Reports, etc.). Proactividad, autonomía y enfoque en resultados.
Colaborar con el equipo de desarrollo para analizar requerimientos y definir criterios de aceptación de calidad. Experiencia con pruebas de API (REST) y herramientas como Postman o Swagger. Experiencia con pruebas de performance o carga. Compromiso con la mejora continua.
Reportar defectos, dar seguimiento y validar correcciones. Conocimientos de CI/CD, Docker y manejo básico de entornos virtualizados.
Asegurar la trazabilidad entre requerimientos, casos de prueba y resultados. Familiaridad con conceptos de testing: smoke, regresión, integración, end-to-end.
Nivel de inglés técnico intermedio.
